## Overview

This guide shows a complete Express.js route handler that receives a Phonelink token and nonce from your frontend, verifies them, and returns the authenticated phone number.

## Install

```bash
npm install phonelink express
```

## Route handler

```ts title="routes/verify-phone.ts"
import { Router } from "express";
import { verifyPhonelinkToken } from "phonelink/server";

const router = Router();

router.post("/api/verify-phone", async (req, res) => {
  const { token, nonce } = req.body;

  if (!token || typeof token !== "string") {
    return res.status(400).json({ error: "Missing or invalid token" });
  }

  if (!nonce || typeof nonce !== "string") {
    return res.status(400).json({ error: "Missing or invalid nonce" });
  }

  try {
    const payload = await verifyPhonelinkToken(
      token,
      nonce,
      process.env.PHONELINK_CLIENT_ID!,
    );

    // At this point, the phone number is cryptographically verified.
    // You can now:
    // - Save it to your database
    // - Associate it with the user's session
    // - Return it to the client

    return res.json({
      phone: payload.phone_e164,
      verified: payload.verified,
    });
  } catch (error) {
    const message = error instanceof Error ? error.message : "Unknown error";

    if (message === "Nonce mismatch") {
      return res.status(401).json({ error: "Nonce mismatch — possible replay attack" });
    }

    if (message === "Phone number not verified") {
      return res.status(401).json({ error: "Phone number not verified" });
    }

    // JWT validation failure (expired, wrong signature, wrong audience, etc.)
    console.error("Token verification failed:", message);
    return res.status(401).json({ error: "Invalid or expired token" });
  }
});

export default router;
```

## Full server example

```ts title="server.ts"
import express from "express";
import verifyPhoneRouter from "./routes/verify-phone";

const app = express();

app.use(express.json());
app.use(verifyPhoneRouter);

app.listen(3000, () => {
  console.log("Server running on http://localhost:3000");
});
```

## Environment variables

```bash title=".env"
PHONELINK_CLIENT_ID=your-client-id
```

## Response codes

| Status | When | Response body |
| --- | --- | --- |
| `200` | Token is valid, phone is verified | `{ phone: "+14155551234", verified: true }` |
| `400` | Missing token or nonce in request body | `{ error: "Missing or invalid token" }` |
| `401` | Token validation failed (any reason) | `{ error: "..." }` describing the failure |

## Adapting for other frameworks

The pattern is the same for any Node.js server framework:

1. Parse `token` and `nonce` from the request body
2. Validate they exist and are strings
3. Call `verifyPhonelinkToken(token, nonce, clientId)`
4. Handle success (use `payload.phone_e164`)
5. Catch errors and return appropriate HTTP status codes
